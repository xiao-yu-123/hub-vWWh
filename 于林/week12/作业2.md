**将历史对话作为大模型的下一次输入**

- 创建会话存储对象：`AdvancedSQLiteSession` 会根据 `session_id` 自动从本地 SQLite 数据库读取该会话的所有历史消息（用户和助手的对话），并将其格式化为大模型所需的 `messages` 列表。

```
# 导入 AdvancedSQLiteSession
from agents.extensions.memory import AdvancedSQLiteSession

# 在 chat 函数内部
session = AdvancedSQLiteSession(
    session_id=session_id,           # 与前端传递的 session_id 关联
    db_path="./assert/conversations.db",  # SQLite 数据库文件
    create_tables=True
)
```

- 将session传递给Runner.run_streamed:`Runner.run_streamed` 内部会调用 `session.get_history()` 获取历史消息，再与当前 `content` 合并构造完整的 `messages` 数组，最终作为大模型的输入。**整个过程对前端透明**，前端只需维护 `session_id`。
